<objective>
Wire the single shared force-state deriver (D-66-01/02) into the list
orchestrator's `installedRowMessage`. Today that function derives `upgradable`
from a pure version-string compare and never resolves; this plan threads the
existing no-network `resolveStrict` cache path into the installed-plugin row so
`force-installed` and `force-upgradable` fall out of one predicate.

This resolves Open Question Q1 by locking the version-differs split documented in
PATTERNS.md: the marketplace cache holds exactly ONE source tree per plugin, so
the split keys on whether the cached tree IS the installed version (versions
equal) or the newer candidate (versions differ).

Purpose: make force-installed/force-upgradable observable on `list`/cascade
surfaces, derived with no persisted flag (FSTAT-01) and no network (FSTAT-05).
Output: an extended `installedRowMessage` returning the new arms plus exhaustive
unit coverage.
</objective>
