## Notes

- `register_authored_at_plan_time: true` -- threats were declared in the PLAN.md `<threat_model>` blocks for Plans 14.1-01 and 14.1-02. This audit verified each declared mitigation by grep + file inspection; the audit does not scan for new vulnerabilities (per workflow contract).
- The consumer-side `PluginCascadeRow` schema in `extensions/pi-claude-marketplace/orchestrators/types.ts:41-42` continues to declare `declaresAgents?: boolean` / `declaresMcp?: boolean` (optional). This is intentional per D-12 (consumer schema unchanged) and does NOT violate T-14.1-06: the **producer** types (`InstallPluginOutcome.installed` at `install.ts:126-127` and `PluginInstalledOutcome` at `execute.ts:64-65`) are REQUIRED `boolean`, so the cascade-row build at `execute.ts:472-473` always passes a defined boolean. The renderer's `composeReasons` defensively accepts `boolean | undefined` (compact-line.ts:445-446) and uses `=== true` checks, which means undefined / false both correctly suppress the marker -- safe under all callers.
- The Rule-3 auto-fix recorded in `14.1-01-SUMMARY.md` (widening the literal `PluginInstalledOutcome` in the reload-hint test) is a typecheck consequence of D-03 and does not introduce a new threat surface.
