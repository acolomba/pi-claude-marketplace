<objective>
Close the CMC-13 gap on the `/claude:plugin import` surface by plumbing `declaresAgents` / `declaresMcp` predicates from the install outcome through the import orchestrator's switch arm and cascade-row construction, replacing the hard-coded `false` literals at `import/execute.ts:474-475`.

Purpose: Per-row soft-dep markers `{requires pi-subagents}` / `{requires pi-mcp}` must fire on `(installed)` rows inside `/claude:plugin import` cascades when the plugin staged the corresponding resource AND the companion extension is unloaded (MSG-SD-1..3). Today they never fire because the import orchestrator hard-codes both predicates to `false`. The renderer is correct; the consumer schema is correct; only the producer wiring is broken.

Output: Typed plumbing change across two source files + the 12 historical test-double widenings AND four new unit tests in `tests/orchestrators/import/execute.test.ts`. No public-API change to the import command's user contract; the only observable difference is that the markers now fire on the import surface as they already do on install / reinstall / update.
</objective>
