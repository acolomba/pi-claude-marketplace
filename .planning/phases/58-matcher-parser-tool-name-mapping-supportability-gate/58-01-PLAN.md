<objective>
Ship the TOOL-01 bidirectional Claude ↔ Pi tool-name map at the
amended location `domain/components/hook-tool-names.ts` (per D-58-04, NOT
`bridges/hooks/tool-names.ts`) plus the paired architecture-test gate that
locks completeness and inverse-invariance.

Purpose: The map is the source of truth Plan 03 (matcher parser) reads at
parse time to translate Claude-form matcher tokens to Pi-form, and Plan 04
(Phase 60 payload translators, future) reads at dispatch time to translate
Pi `event.toolName` to Claude `tool_name`. Locking it as a closed-set
table with compile-time exhaustiveness against the peer-dep
`ToolCallEvent.toolName` union prevents Pitfall-10 silent mapping drift
when the peer-dep adds a tool.

Output: One new domain-tier source file + one new architecture-test file.
No source-rename, no closed-set rename, no other surface mutation.
