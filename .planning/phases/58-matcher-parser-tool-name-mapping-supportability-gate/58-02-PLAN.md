<objective>
Ship the bucket-A 8-event closed-set tuple + per-non-tool-event field-name
+ per-event Claude-side closed value-set tables at
`domain/components/hook-events.ts` (sibling to Plan 01's `hook-tool-names.ts`,
per D-58-06 + CONTEXT.md "Claude's Discretion"). Plus the paired
architecture-test gate that locks the tuple shape and per-event closed-set
membership.

Purpose: These tables are the source of truth Plan 03's
`checkMatcherSupportability` reads to trip TOOL-02(c) when:
(i) the plugin declares hooks under any non-bucket-A event key, or
(ii) a non-tool event matcher contains a Claude `source`/`reason` value
that has no Pi peer-dep analog (strict-supportability stance over silent
never-fire), or (iii) a UserPromptSubmit matcher is non-empty (Claude has no
matcher support upstream — strict-supportability over silent ignore).

The Pi peer-dep verification done in 58-RESEARCH.md identified concrete
gaps: SessionStart Pi `reason` is `"startup" | "reload" | "new" | "resume" | "fork"`
(no `clear` / `compact`); SessionShutdown Pi `reason` is
`"quit" | "reload" | "new" | "resume" | "fork"` (most Claude
SessionEnd values unmappable); PreCompact / PostCompact carry no
`trigger` field. The strict-supportability stance per D-58-06 means
these become TOOL-02 trips at parse time.

Output: One new domain-tier source file + one new architecture-test file
scaffold (further-populated by Plan 03 with the parse-time
TOOL-02 invariants).

  (i.e. have a Pi peer-dep analog). UserPromptSubmit has no entry
  (sentinel). PreCompact/PostCompact have empty sets (no Pi trigger field
  → no admissible non-empty matcher value).
