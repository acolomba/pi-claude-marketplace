### CR-01 cascade-block-ordering remediation

- **D-01:** Replace the 3 local `function scopeOrder` helpers in `orchestrators/plugin/reinstall.ts:706-708`, `orchestrators/plugin/update.ts:1062-1064`, and `orchestrators/import/execute.ts:700-702` with the canonical `compareByNameThenScope` from `presentation/sort.ts:51`. Delete the local helper declarations entirely once unused.
- **D-02:** Fix `orchestrators/marketplace/autoupdate.ts:114` by flipping the iteration order from `["user", "project"]` to `["project", "user"]` so the actual user-visible row order matches the existing comment at lines 174-178 and aligns with MSG-GR-3 / `compareByNameThenScope` project-first policy across all surfaces. (The autoupdate result block uses a name-only `localeCompare` sort, so scope iteration order only matters for same-marketplace-name-cross-scope ties.)

### MSG-GR-3 rule tightening

- **D-05:** Replace `tests/lint-rules/msg-gr-3-per-scope.js` no-op `Program: () => {}` visitor with an active AST check that detects two patterns: (a) any function declaration named `scopeOrder` (or arrow-shaped equivalent) returning a numeric where the `"user"` branch evaluates to a lower value than `"project"`; (b) any array literal `["user", "project"]` (or `as const` variant) in files under `orchestrators/`. Both patterns together cover the local-helper drift and the iteration-order drift seen in autoupdate.ts.
- **D-06:** Apply the tightened rule to `extensions/pi-claude-marketplace/orchestrators/**/*.ts` via the existing `eslint.config.js` per-rule `files:` block. Mirrors how the other MSG-* rules are scoped. Avoids touching `presentation/sort.ts` (the canonical comparator) so no allowlist is needed.
- **D-07:** Update `tests/architecture/msg-rule-registry.test.ts` parity fixtures + `docs/messaging-style-guide.md` MSG-GR-3 frontmatter description: change wording from "structural meta-assertion" to "active AST lint check" so the 4-way registry parity (rule file ↔ eslint.config ↔ frontmatter ↔ style guide) stays green.
- **D-08:** Record a decision entry against D-14-09 LOCKED in PROJECT.md noting that MSG-GR-3's classification has moved from meta-assertion (D-14-09) to active lint check; the structural enforcement claim is upgraded, not weakened.
