<decisions_made>

- Implemented reinstall as a dedicated single-plugin core because install rejects existing records, uninstall removes old resources first, and update imports Git/network refresh machinery with weaker recovery semantics.
- Used `withLockedStateTransaction` instead of `withStateGuard` so physical resources can be rolled back if explicit state persistence fails.
- Kept reinstall version policy pinned to `oldRecord.version`; the orchestrator does not call `resolvePluginVersion` or compute content hashes.
- Used a self-exempt state clone for `assertNoCrossPluginConflicts` so unchanged generated names do not conflict with the target plugin's own current record.
- Kept MCP collision policy inside the MCP bridge and force/foreign-agent policy inside the agents bridge.
- Added test-only `__deps` injection seams for state transaction, cache invalidation, and data cleanup failures to keep rollback/cleanup tests deterministic without production-only hacks.
</decisions_made>
