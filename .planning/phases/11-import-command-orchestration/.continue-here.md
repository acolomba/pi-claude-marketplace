<decisions_made>

- Added an optional notification/outcome seam to `installPlugin` instead of duplicating install internals. Rationale: import needs typed outcomes and single reload aggregation while normal installs must preserve current behavior.
- `importClaudeSettings` uses production defaults for settings/state/add/install but accepts injectable deps for tests. Rationale: this allows fast, deterministic orchestration tests without bypassing production delegation paths.
- Source mismatch blocks only dependent plugin imports for that marketplace/scope. Rationale: avoids silently installing from a different Pi marketplace than Claude settings requested while preserving safe continuation for unrelated imports.
- Edge handler uses `parseArgs` directly because import has no positionals and omitted scope intentionally expands to both scopes. Rationale: existing command convention supports `--scope user|project`; import is the special case where omitted scope is both rather than user.
</decisions_made>
