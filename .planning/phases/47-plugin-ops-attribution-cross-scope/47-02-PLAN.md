<objective>
Re-attribute reinstall's marketplace-existence / scope precondition to the canonical `info`
`(failed) {not added}` form, consistently across the explicit-scope-plugin, explicit-scope-marketplace,
and bare forms (the audit A-4 "self-inconsistent across forms" finding), and tighten the
`narrowReason` substring fallback so a missed cascade/IO failure never lies `{not in manifest}`.

This plan closes ATTR-03, the reinstall half of ATTR-09, and the reinstall half of SCOPE-01.

Purpose: today reinstall reports the SAME marketplace-absent condition three different ways --
`(skipped) {not installed}` for an explicit-scope plugin reinstall (synthesized phantom target →
runLockedReinstall M6), and `(failed) {not found}` for the explicit-scope-marketplace and bare forms
(raw `MarketplaceNotFoundError`/`Error` → reinstallPlugins catch → synthetic `(reinstall)` row M7/M8).
Phase 47 makes all three emit one standalone `{not added}` (D-47-A) BEFORE any cascade, matching
`info` and the install/uninstall plan (47-01).

Output: amended `reinstall.ts` (structural marketplace-absent signal + entrypoint standalone
emission + tightened `narrowReason`), with paired catalog states + fixtures. Independently GREEN
(`npm run check` exit 0).

This plan is part of the SERIALIZED single-wave phase: it depends on Plan 47-01 (which lands the
shared discriminated cross-scope resolver and the catalog template) and runs strictly after it.
It re-touches `docs/output-catalog.md` + `tests/architecture/catalog-uat.test.ts` (added states
only -- no overlap with 47-01's install/uninstall states).
</objective>
