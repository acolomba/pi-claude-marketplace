Purpose: lifecycle hardening. Without WR-03, a plugin installed at runtime via the standalone `install` command would not fire its hooks until the next `/reload` (because the routing table was last rebuilt during the previous load's `apply.ts` cascade and is not refreshed by the standalone install flow). With Plan 60-02's exec body in place, this becomes correctness-critical the moment a real plugin author exercises the install command outside a /reload cycle. WR-01 is bounded info-disclosure (phantom user-scope entries surfacing under the wrong cwd at factory time) — co-located to keep the lifecycle scope tight.

- 60-03 modifies `dispatch.ts` + adds `event-adapters.ts`.
