<objective>
Wire the parse-time-compile path for the `if` field: extend
`HOOK_HANDLER_SCHEMA` with `if: Type.Optional(Type.String())`, add
`compileIfPredicate(rawIf, claudeEvent, ctx)` to
`domain/components/hooks.ts`, extend `parseHooksConfig` to walk every
(event, group, handler) triple producing a side-Map of compiled
predicates, extend `RoutingEntry` with `ifPredicate: IfPredicate`, and
populate the field in `flattenPluginIntoBuckets`. Every failure path
collapses to `MATCH_ALL_IF` per D-61-02; the plugin always installs.

Purpose: Plan 01 shipped the pure-data primitives (glob engine, Bash
parser, target table, discriminated union, fall-open sentinel). Plan 02
makes them observable on every `RoutingEntry` so Plan 03's
single-line dispatch consult has a typed field to read. The plan never
modifies the dispatch hot path; all expensive work happens at
registration time (`parseHooksConfig` or `flattenPluginIntoBuckets`),
mirroring Phase 58's registration-time-translation stance.

`if-field/index.ts`, plus 4 call-site sources (`resolver.ts`,
`reinstall.ts`, `install.ts`, `update.ts`). Tests: `hooks-if-field.test.ts`,
`hooks.test.ts`, `update.test.ts`, `uninstall.test.ts`. Net ~150-250 lines
added; Plan 03's dispatch wire is a 1-line insertion.
