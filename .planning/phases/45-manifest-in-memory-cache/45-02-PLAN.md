Two file changes realize D-01..D-04 exactly as the POC proved:
1. NEW `domain/manifest-cache.ts` -- `createManifestCache(loader)` factory owning its
   own `Map<string, Entry>` keyed by `manifestPath`, each entry holding
   `(mtimeMs, size, ok, value?, error?)`. Per-`load`: one `stat`; on a `(mtimeMs,size)`
   match serve by reference (success) or re-throw the same Error (negative); on
   miss/change/stat-fail fall through to the injected loader and refresh the entry.
   The cache adds ONLY a `stat`, never a `readFile` (CACHE-06).

   `loadMarketplaceManifestUncached` (the sole `readFile(...marketplace.json)`,
   returning the RAW `JSON.parse` value), construct one module-level singleton
   `createManifestCache(loadMarketplaceManifestUncached)`, and make the exported
   `loadMarketplaceManifest` delegate to `manifestCache.load(manifestPath)`.

re-validate cost -- with byte-identical output and zero call-site churn (all 9
callers consume the result read-only, D-03).
Output: `domain/manifest-cache.ts` (new) + delegating `domain/manifest.ts`; Wave 0
