<objective>
Land the new failure-class reason token `{malformed mcp}` (D-02) in the closed-set
reason catalog and route the resolver's `malformed mcp reference:` notes to it,
without reclassifying the existing inline `malformed mcpServers` notes (which must
stay `{unsupported source}`).

Purpose: Give MCPR-03's per-plugin degrade a truthful user-facing token. A broken
`mcpServers` STRING reference is a malformation of a SUPPORTED feature the resolver
actively parses — semantically failure-class (parallel to `{invalid manifest}` /
`{unparseable}`), NOT the "unsupported KIND" family. This plan depends on Plan 01,
which emits the `malformed mcp reference:` notes this narrower matches.

Output: `malformed mcp` appended to `REASONS` and `FAILURE_REASONS`; a new
prefix-anchored branch in `narrowResolverNotes` with a widened local return alias;
the closed-set length tripwire bumped 34 -> 35; a collision-guard test (both
directions); and the `docs/output-catalog.md` reason-vocabulary + `(unavailable)`
row update.
</objective>
