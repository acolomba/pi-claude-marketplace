<objective>
Extend the on-disk state schema so a `resources.hooks` field exists as a required `string[]` on every plugin install record without bumping `schemaVersion`. v1.12 state.json files must load cleanly and gain `resources.hooks: []` defaults via the existing default-fill seam (`ensurePluginResources`) before validation runs.

Purpose: Per D-57-01 (HOOK-02 amendment), the migration is purely additive because no v1.0–v1.12 state.json record carries hook resources (every hook-using plugin was rejected as `UNSUPPORTED_COMPONENT_KIND`). A `schemaVersion` widening would be signaling-only ceremony; mirroring the existing `agents: []` / `mcpServers: []` default-fill pattern is the smaller and load-bearing diff.

Output: Updated `PLUGIN_INSTALL_RECORD_SCHEMA`, extended `ensurePluginResources`, two new test cases covering the default-fill and the v1.12 round-trip.
</objective>
