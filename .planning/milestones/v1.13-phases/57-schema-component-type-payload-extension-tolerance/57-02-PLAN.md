<objective>
Introduce the forward-compatible TypeBox schema for Claude `hooks.json` files. The schema accepts any string at the top-level event-key position (D-57-02), permits unknown fields at every nesting level (HOOK-03), and rejects only JSON syntax errors and structural shape mismatches (D-57-04). Expose a compiled validator and a `parseHooksConfig` helper returning a discriminated `{ ok: true, value } | { ok: false, reason }` result so the resolver (Plan 03) can hook parse failure into the `installable: false` cascade.

Purpose: Per HOOK-03, every Claude hook-config field outside Claude's documented schema must be silently accepted (no schema-level gate) so v1.14+ event-set promotions and Claude-Code field additions never force a downstream version-bump cascade. Per D-57-02, the bucket-A admission gate lives in Phase 58's TOOL-02(c), NOT in this schema. Per D-57-04, JSON-syntactic / structural / required-field-on-`type: "command"` failures flip the plugin to `installable: false` via the resolver in Plan 03.

Output: New `domain/components/hooks.ts` module exporting `HOOKS_CONFIG_SCHEMA`, `HOOKS_VALIDATOR`, and `parseHooksConfig`; matching behavior tests in `tests/domain/components/hooks.test.ts`.
</objective>
