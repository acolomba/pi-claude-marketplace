Purpose: Phase 59's central deliverable is the routing layer. The router
owns three pieces of module-level state (liveEpoch, parsedConfigCache,
routingTable), exposes a small public API (5 functions + 1 type), and
captures the 7 distinct Pi-event registrations as a unit. Building it as
its own plan — separate from the wiring — keeps the architecture-tests
narrow and the review surface contained. The dispatchHookExec stub
preserves the Phase 60 hand-off: the composite handlers don't change
signatures when execution lands; only the stub's body grows.

Output: 5 new files (4 in bridges/hooks/, 1 test pair), 2 modified files
(bridges/index.ts barrel + platform/pi-api.ts re-exports), unit-tested
to pin cache, rebuild, ordering, and epoch contracts.
