<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| pluginRoot → bridge writer | Untrusted plugin payload (filesystem entries under `<pluginRoot>/hooks/`) crosses into the bridge — a hostile or careless plugin may ship a symlinked-escape inside the hooks subtree. |
| bridge writer → scopeRoot disk | The bridge writes to `<scopeRoot>/pi-claude-marketplace/hooks/<plugin>/hooks.json` — write target MUST stay inside `hooksDir` (NFR-10). |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-63-LIFE-03-SE | Tampering | `<pluginRoot>/hooks/` subtree | mitigate | `assertNoSymlinkEscapeInHooksSubtree(pluginRoot)` calls `readdir({recursive: true, withFileTypes: true})`, then for each symlink entry calls `fs.realpath` + `assertPathInside(pluginRoot, realpath, label)`. First escape throws `SymlinkRefusedError` (subclass of `PathContainmentError`) which propagates through PI-14 instance-check handling. |
| T-63-02-NAME | Tampering | plugin name on write path | mitigate | `assertSafeName(plugin, "hooks bridge plugin name")` runs BEFORE `path.join(hooksDir, plugin, "hooks.json")` — rejects `/`, `\`, `.`, `..`, ASCII control chars. |
| T-63-02-CONT | Tampering | hooks.json write target | mitigate | `atomicWriteJson` writes the leaf only; an extra `assertPathInside(hooksDir, hookConfigPathFor(locations, plugin), ...)` guard runs before the write call as belt-and-suspenders (the constructed path is deterministic but the assert pins the invariant in the test). |
| T-63-02-PW | Tampering | partial-write of `<plugin>/hooks.json` corrupting state | mitigate | `write-file-atomic@^8` via `atomicWriteJson` writes to tmp + fsync + rename atomically. NFR-1 satisfied. |
| T-63-02-TOCTOU | Tampering | TOCTOU between symlink-check and atomic write | accept | Residual risk acknowledged in `shared/path-safety.ts` comment block; threat model is "careless or malicious plugin author", not "concurrent in-process attacker". Acceptable for v1.13. |
</threat_model>
