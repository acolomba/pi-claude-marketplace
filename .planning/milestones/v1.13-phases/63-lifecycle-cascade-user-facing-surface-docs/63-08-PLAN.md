<objective>
Close the CR-01 gap on Truth #2 / LIFE-03 by tightening the symlink-escape walker in `extensions/pi-claude-marketplace/bridges/hooks/stage.ts` so it NEVER enumerates paths outside `<pluginRoot>/hooks/`. The current implementation at lines 55-107 uses `readdir(hooksRoot, { recursive: true, withFileTypes: true })`, and Node's recursive readdir follows symlinks to directories â€” so a symlinked-escape directory is enumerated (uncontained external-tree FS reads occur) BEFORE the walker reaches and rejects the symlink entry. The fix is a hand-rolled stack walk that calls `lstat` at each entry and refuses to descend through any entry whose `lstat` reports `isSymbolicLink()`.

Purpose: LIFE-03's stated rejection contract (`SymlinkRefusedError` raised; `fs.realpath` + `assertPathInside(pluginRoot, realpath, ...)` containment check) is technically satisfied today â€” the test suite passes â€” but its INTENT (refuse to read outside pluginRoot) is violated. The threat model in `63-02-PLAN.md::T-63-LIFE-03-SE` says "first escape throws `SymlinkRefusedError`" but does not address the pre-rejection reads. This plan tightens the walker shape so the threat is fully contained and adds regression assertions that pin both the rejection message subject and the absence of external-tree FS activity.

Output: Replaced internal body of `assertNoSymlinkEscapeInHooksSubtree` (no signature change, no exported-symbol change); tightened regression assertions in `tests/bridges/hooks/symlink-escape.test.ts`; corrected schema-valid `HOOKS_VALUE` fixtures in the two bridge test files (WR-05 co-located cleanup).
