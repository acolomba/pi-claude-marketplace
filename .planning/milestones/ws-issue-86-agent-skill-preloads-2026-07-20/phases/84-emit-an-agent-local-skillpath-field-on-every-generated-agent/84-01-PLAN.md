<objective>
Deliver the code-side heart of AGSK-06: emit an agent-local `skillPath` on
every generated agent that references skills, and collapse the body skill
legend to a single `(available on demand)` state (re-amending AGSK-04). This
plan changes the emitter (`frontmatter.ts`) and its legend producer
(`convert.ts`) together, then updates the byte-identity unit-fixture corpus so
`node --test` stays green.

Purpose: The conversion pipeline already emits `skills:` correctly, but the
emitted names never resolve for the spawned subagent because the skill installs
off pi-subagents' scan roots. A fixed-constant `skillPath` pointer fixes
resolution (SC-1). Under pi-subagents 0.35.x lazy delivery nothing is eagerly
preloaded, so the two-state legend overclaims — D-84-01 collapses it.

Output: `frontmatter.ts` emits the skillPath line and renders a single-state
legend; `convert.ts` no longer computes the retired per-entry flag; both unit
test files pin the new bytes.
</objective>
