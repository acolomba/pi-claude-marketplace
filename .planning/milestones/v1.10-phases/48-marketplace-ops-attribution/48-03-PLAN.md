`InvalidMarketplaceManifestError`. ATTR-10 teaches `reasonsFromCascadeError` to recognize it and map
to `["invalid manifest"]` (Decision D-48-B Option B2, the typed-dispatch convention), so the
`?? network unreachable` default no longer fires for a path-source manifest failure. github-source
failures with no errno KEEP `network unreachable` as the catch-all (Pitfall 3: no classification
collapse).

marketplace states (`failure-unreachable` add, `mp-failure-network` update, the autoupdate bare
not-found IF any remains) must be byte-identical after the D-48-A `MpFailed.reasons?` addition
(reasons omitted → brace collapses), and serves as the final phase GREEN gate.

NFR-5 confirmation (RESEARCH D-48-B): the path-source branch of refreshRecord (update.ts:387-388)
calls only validateManifestAtRoot → loadMarketplaceManifest (readFile + parse). NO gitOps, NO
platform/git. The fix is reason-only; the path code path was already network-free.
