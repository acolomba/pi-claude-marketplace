<objective>
Re-attribute update's marketplace-existence / scope precondition (both the `<plugin>@<mp>` and the
`@<mp>` marketplace forms) to the canonical `info` `(failed) {not added}` form, eliminate the raw
`MarketplaceNotFoundError`/`Error` that escapes to the `{not found}` synthetic row, and preserve the
cascade path's never-throw contract.

This plan closes ATTR-02 and the update half of SCOPE-01. It is the final plan of the serialized
single-wave Phase 47.

Purpose: today the direct update path reports a missing marketplace as `(failed) {not found}` --
`enumerateMarketplaceTarget` raw `Error` (M10), `resolveInstalledMarketplaceTarget` raw
`MarketplaceNotFoundError` (M11), and `syncCloneOnce` raw `Error` -- all caught by `updatePlugins` →
`notifyDirectFailure` → `narrowDirectFailReason` → `{not found}`. `preflightUpdate`'s `mp === undefined`
also returns `(skipped) {not in manifest}` (M9), but `preflightUpdate` is SHARED by the cascade path
(`updateSinglePlugin`, which must NEVER throw) -- so the fix hoists the marketplace-existence check to
the DIRECT path entrypoint (where M10/M11 live) and emits the standalone variant there, leaving the
cascade-safe `preflightUpdate` outcome as the concurrent-removal guard (RESEARCH Pitfall 3 / Open
Question #2 / Assumption A3).

Output: amended `update.ts` (structural mp-absent signal + direct-path standalone emission) and
`shared.ts` (`resolveInstalledMarketplaceTarget` returns a structural not-found result instead of a
raw throw), with paired catalog states + fixtures. Independently GREEN (`npm run check` exit 0).

This plan depends on Plan 47-02 (and transitively 47-01) and runs strictly last in the serialized
wave -- it is the final `shared.ts` touch (migrating `resolveInstalledMarketplaceTarget` to the
discriminated result and retiring the now-dead raw-throw paths in reinstall/update).
</objective>
