## Integration & Flows

Single-phase milestone — no cross-phase wiring seams. The one E2E flow (install a
plugin whose `mcpServers` is a `./`-relative string → the referenced servers stage
after `/reload`, at parity with the inline form) is verified by construction: the
new string branch in `applyStrictMcp` feeds the resolved server map into the
**unchanged** `applyMcpValue` → `partial.mcpServers` path, so all downstream MCP
staging is byte-identical to the already-shipped inline form.

Cross-**surface** parity (list / info / install all render `{malformed mcp}` for a
broken reference) was a real risk — the install surface uses a hand-mirrored
classifier (`install.ts::narrowResolverReasons`) that the initial implementation
missed. Code review caught it (CR-01 BLOCKER), the fix mirrored the arm, and a
dedicated cross-surface parity test (`tests/orchestrators/plugin/cross-surface-reason-parity.test.ts`,
18/18) now locks it. The phase verifier independently re-confirmed all surfaces.
