<objective>
Land the shared spine that every command migration in Wave 2 builds against, with
zero rendered-output change. This plan creates the horizontal surface (the
`CommandContext` type contract + shared naming convention), exports the central
presentation vocabulary so sibling modules can call it, splits the closed reasons
set into topic-grouped enums (membership preserved), adds the inert optional
caller-intent base fields, scaffolds tuple-vs-array cardinality typing, and
introduces a `notifyWithContext` entry point that delegates to the existing central
render/severity/summary/reload path (adapter-then-migrate, per research A1).

Purpose: Decouple the 18 command migrations from each other and from the legacy
`notify()` signature. The spine is additive — the legacy `notify(ctx, pi, message)`
keeps working unchanged so `catalog-uat` (114 fixtures) and `notify-v2` stay green
while commands migrate one family at a time in Wave 2.

Output: `shared/notify-context.ts` (new), `shared/notify-reasons.ts` (new), and an
additive set of exports + the `notifyWithContext` adapter in `shared/notify.ts`.
This plan implements MOD-01/MOD-02/MOD-03 scaffolding, OUT-07 cardinality typing,
and the OUT-08 closed-set preservation.
</objective>
