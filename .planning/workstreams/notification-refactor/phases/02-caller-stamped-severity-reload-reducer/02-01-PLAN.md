Turn the inert Phase-1 `severity`/`needsReload` fields LIVE: every transition (state-change) producer stamps both fields, the 5 transition message interfaces narrow them to REQUIRED (so omission is a compile error), and `notify()` becomes a dumb reducer — emission severity = numeric MAX over `row.severity`, `/reload` trailer = OR-reduce of `row.needsReload`, tallies by stamped facts — with the content-derived ladders (`BENIGN_REASONS`/`allBenign`/`cascadeSeverity`/`reconcileAppliedSeverity`/`shouldEmitReloadHint` token mapping) deleted. Purpose: relocate severity/reload correctness from one central reducer to the producers (SEV-01..05, RLD-01..03) with GATE-01 type-level enforcement, while keeping output BYTE-IDENTICAL (D-01).
Output: live stamped fields, narrowed transition interfaces, a dumb reducer, deleted content ladders, repaired tests — all gated green by catalog-uat byte-equality + tsc + npm run check.
</objective>
