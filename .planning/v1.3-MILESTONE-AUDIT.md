## CR-01: Cascade-Block-Ordering Drift (WARNING)

**Files (3 orchestrators + 1 supporting handler all share the bug):**
- `extensions/pi-claude-marketplace/orchestrators/plugin/reinstall.ts:457-464, 706-708`
- `extensions/pi-claude-marketplace/orchestrators/plugin/update.ts:795-804, 1062-1064`
- `extensions/pi-claude-marketplace/orchestrators/import/execute.ts:347-374, 700-702`
- `extensions/pi-claude-marketplace/orchestrators/marketplace/autoupdate.ts:114, 174-178` (corroborating: `scopes = ["user", "project"]` iteration plus stable `Array.prototype.sort` → user-first on same-name ties)

**Issue:** Each of reinstall / update / import/execute defines a local helper `function scopeOrder(scope: Scope): number { return scope === "user" ? 0 : 1; }` and uses it to sort cascade marketplace blocks **user-before-project**. The JSDoc/comment at every callsite explicitly claims "project before user per `compareByNameThenScope`" — but the implementation does the opposite. The canonical `compareByNameThenScope` at `presentation/sort.ts:51` returns **project-first** and is used by `marketplace-list.ts`, `plugin-list.ts`, `cascade-summary.ts` (within-block rows), and `orchestrators/plugin/list.ts`.

**Cross-surface observable break:** When a user has the same-named marketplace in both scopes:
- `/claude:plugin list`, `/claude:plugin marketplace list` render **project-first**.
- `/claude:plugin reinstall`, `/claude:plugin update`, `/claude:plugin import` cascades render **user-first**.

**Why it wasn't caught:**
- `catalog-uat.test.ts` fixtures don't exercise same-marketplace-name-cross-scope cases (`local-mp` vs `official` sort alphabetically by name, so the scope tiebreaker never fires)
- The MSG-GR-3 ESLint rule at `tests/lint-rules/msg-gr-3-per-scope.js:50` is a no-op meta-assertion that cites structural enforcement but does not lint for the inverse anti-pattern

**Affected requirements:** CMC-03 (single per-scope rendering policy), CMC-25, CMC-26, CMC-27. All marked Complete in REQUIREMENTS.md but the inverse drift exists.

**Suggested remediation (outside Phase 14 scope):** Replace the 3 local `scopeOrder` helpers with `compareByNameThenScope`; update `tests/orchestrators/plugin/reinstall.test.ts:984-988` fixture; fix misleading comment at `autoupdate.ts:114, 174-178`. Single closure phase covering all 3 orchestrators would also tighten the MSG-GR-3 rule from no-op meta-assertion to an actual lint check that detects local user-first comparators.
