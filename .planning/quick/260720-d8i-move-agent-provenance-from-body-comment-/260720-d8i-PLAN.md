<objective>
Move generated-agent provenance out of the body HTML comment and into flat
frontmatter keys, so the bridged subagent's LLM (which reads the file body
verbatim as its system prompt) no longer ingests provenance text.

Purpose: pi-subagents sets `systemPrompt: body` verbatim with no comment
stripping. Provenance currently lives in a body HTML comment, so the child LLM
reads it. pi-subagents' frontmatter parser stores every `key: value` line in a
flat record and silently ignores keys it does not know, so provenance rendered
as extra frontmatter keys is parsed-and-ignored and never reaches the LLM.

Output: an updated emitter, retargeted parser-safety helper, corrected doc
comments, regenerated byte-pin tests, and a CHANGELOG entry. `npm run check`
green.
</objective>

#    comment stripping) -- so the body IS the system prompt.
# 2. pi-subagents' frontmatter parser matches each line against
#    ^([\w-]+):\s*(.*)$, stores ALL key:value pairs into a flat record, strips
#    one surrounding matching quote pair, and SILENTLY IGNORES unknown keys and
#    any non-matching line. Consumers read only the keys they know

#    contents.includes(GENERATED_AGENT_MARKER)). So provenance LOCATION in the
