  install orchestrator already imports `PathContainmentError` directly
  (`orchestrators/plugin/install.ts:117`). Both modules are in the
  `orchestrators/` -> `shared/` import direction (import-x layering allows
  this). Use `assertPathInside` directly from `derivePluginRootForInfo` and
  let `PathContainmentError` propagate -- it is already an `Error` subclass,
  so `narrowProbeError` classifies it via the `code === undefined` fallback

  token. Do NOT extend `narrowProbeError` with a containment-specific arm in
  this pass; the `unreadable` fallback is the intentional generic bucket.
